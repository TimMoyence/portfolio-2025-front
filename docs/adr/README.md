# ADR Frontend

Les ADR servent a garder une trace courte, lisible et datee des choix d'architecture.

## Regles

- Nommer les fichiers `NNNN-titre-court.md`
- Garder un seul sujet par ADR
- Mettre a jour le statut si une decision est remplacee
- Lier l'ADR depuis le README ou la doc de gouvernance si le sujet impacte l'onboarding
